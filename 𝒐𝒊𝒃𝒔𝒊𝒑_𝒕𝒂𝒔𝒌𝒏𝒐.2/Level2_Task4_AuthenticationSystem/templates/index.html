About Project

Project: Basic Login Authentication System

  1. Project Overview:

    This is a simple login authentication system using Flask(a Python microframework), that allows users to register, login, and access a secured page.

  2. Features:
    • The system has a user registration feature where users can sign up by providing a useranme and password as login credentials and other additional informations.
    • The system stores user information securely(passwords are encrypted).
    • The system has a login feature where users can enter correct username and password to access the secured page or content.
    • The secured page is only accessible to the logged in users.
  3. Tools and Technologies:
      The system is developed on Python programming language implementing Flask(Python Microframework).
      All the user data are being stored in Sqlite3 database.
  4. Implementation Steps:
    1. User Registration:
      • User is prompted to register by providing - first name, last name, email, username and password.
      • Upon form submissions the input data will be compared and searched if there is any user with same username or email. If any existing user is found, the user will be returned back to the registration page, or the user can simply login.
    2. Password Encryption:
      • The password created by the user will be encrypted for security puposes.
      • All the user data then is stored into Sqlite3 database.
    3. User Login:
      • After successful registration the user will be prompted to login page. User can login by enterring correct username and password.
      • The entered username is found valid, the entered password is compared with the stored encrypted password. If they match, the user gets logged in.
    4. Secured Page:
      • Unauthorised users cannot access the page content.
      • Once user is logged in, the login and signup pages will be inaccessible.
  5. Error Handling:
    • Implemented proper error handling to handle cases like incorrect login credentials, already registered users, etc.
  6. Testing:
    • This system is throughly tested to ensure that it works as expected, covering both positive and negative scenarios.
